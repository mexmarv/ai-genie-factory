# Databricks App Design ‚Äî Alpura UX/UI Skill

This skill governs the visual design and UX patterns for all Databricks Apps generated by the
AI Genie Factory at Alpura. Read it before writing or reviewing any `ui.py` or `app.py` file.

## Platform Context

All apps run inside **Databricks Apps** (Dash-based) or optionally **Streamlit on Databricks**.
The host environment is dark-themed. Apps must feel native to the Databricks workspace ‚Äî not like
standalone web apps dropped in.

**Default rendering target:** Databricks Apps (Dash)
**Fallback:** Streamlit on Databricks (same design tokens apply)

---

## Design Tokens

These are the only approved values. Never invent new colors or fonts per app.

## Component Patterns

### KPI Card

```python
# ui.py ‚Äî KPI card pattern
def kpi_card(label: str, value: str, delta: str = None, delta_positive: bool = True) -> dict:
    """Returns a Dash html.Div KPI card."""
    delta_color = "--accent-green" if delta_positive else "--accent-red"
    delta_el = html.Div(delta, style={"color": f"var({delta_color})", "fontSize": "12px"}) if delta else None
    return html.Div([
        html.Div(label, style={"color": "var(--text-secondary)", "fontSize": "12px", "marginBottom": "4px"}),
        html.Div(value, style={"color": "var(--text-primary)", "fontSize": "36px", "fontWeight": "700", "lineHeight": "1"}),
        delta_el
    ], style={
        "background": "var(--bg-card)",
        "border": "1px solid var(--border)",
        "borderRadius": "8px",
        "padding": "16px",
        "flex": "1"
    })
```

### Plotly Chart Container

All charts must use these base settings ‚Äî never override the template:

```python
# ui.py ‚Äî base chart layout applied to every figure
BASE_LAYOUT = dict(
    template="plotly_dark",
    paper_bgcolor="var(--bg-card)",
    plot_bgcolor="var(--bg-card)",
    font=dict(family="Inter, system-ui", color="var(--text-secondary)", size=11),
    margin=dict(l=40, r=20, t=40, b=40),
    title_font=dict(size=13, color="var(--text-primary)", family="Inter, system-ui"),
    legend=dict(bgcolor="rgba(0,0,0,0)", bordercolor="var(--border)"),
    xaxis=dict(gridcolor="var(--border)", linecolor="var(--border)"),
    yaxis=dict(gridcolor="var(--border)", linecolor="var(--border)"),
)

def apply_base_layout(fig) -> go.Figure:
    fig.update_layout(**BASE_LAYOUT)
    return fig
```

### Filter Bar

```python
# ui.py ‚Äî standard date range filter
def date_filter(id_prefix: str, default_days: int = 30) -> html.Div:
    end = datetime.today()
    start = end - timedelta(days=default_days)
    return html.Div([
        html.Label("Date Range", style={"color": "var(--text-secondary)", "fontSize": "12px"}),
        dcc.DatePickerRange(
            id=f"{id_prefix}-date-range",
            start_date=start.date(),
            end_date=end.date(),
            style={"background": "var(--bg-elevated)"}
        )
    ], style={"display": "flex", "flexDirection": "column", "gap": "4px"})
```

## Chart Type Rules

| Data shape | Chart type | Notes |
|---|---|---|
| Trend over time | `px.line` with fill | `fill="tozeroy"`, accent-blue |
| Category comparison | `px.bar` horizontal | Sort descending, accent-blue |
| Part of whole | `px.treemap` | Avoid pie charts ‚Äî hard to read on dark bg |
| Distribution | `px.histogram` | accent-blue bars, border color |
| Geo / region | `px.choropleth_mapbox` | Only when data is genuinely geographic |
| Correlation | `px.scatter` | accent-blue dots, trendline optional |
| Multi-metric | `go.Figure` with subplots | Use `make_subplots`, share x-axis for time series |

Never use 3D charts ‚Äî they are unreadable in dashboards.
Never use `px.pie` ‚Äî use `px.treemap` instead.

## Delta / Change Indicators

When showing period-over-period change:

```python
def format_delta(current: float, previous: float) -> tuple[str, bool]:
    """Returns (formatted_delta_string, is_positive)"""
    if previous == 0:
        return "N/A", True
    pct = ((current - previous) / previous) * 100
    sign = "‚ñ≤" if pct >= 0 else "‚ñº"
    return f"{sign} {abs(pct):.1f}% vs prior period", pct >= 0
```

## Number Formatting

```python
def fmt_number(n: float, decimals: int = 0) -> str:
    """Format large numbers with K/M/B suffixes for KPI cards."""
    if n >= 1_000_000_000:
        return f"{n/1_000_000_000:.1f}B"
    if n >= 1_000_000:
        return f"{n/1_000_000:.1f}M"
    if n >= 1_000:
        return f"{n/1_000:.1f}K"
    return f"{n:,.{decimals}f}"

def fmt_currency(n: float) -> str:
    return f"${fmt_number(n)}"

def fmt_pct(n: float) -> str:
    return f"{n:.1f}%"
```

## What NOT to Do

- No light backgrounds ‚Äî the workspace is dark, light apps break the visual context
- No rainbow color schemes ‚Äî use the token sequences only
- No more than 4 KPI cards in a row ‚Äî use two rows if needed
- No full-width single-column chart layouts ‚Äî always 2-up or intentionally full-width with justification
- No `px.pie` ‚Äî use `px.treemap`
- No inline styles with raw hex values ‚Äî always reference design tokens
- No chart titles in ALL CAPS ‚Äî sentence case only
- No axis labels that say "value" or "count" ‚Äî always name the metric explicitly
- No default Plotly blue (#636efa) ‚Äî use `--accent-blue` (#00bcd4)
